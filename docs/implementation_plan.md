# План разработки на 3 дня

## День 1 — База backend и БД

1. Подготовить инфраструктуру проекта (`backend`, `frontend`, `worker`, `bot`, `docker-compose`, `.env.example`).
2. Поднять backend (FastAPI) с endpoint'ами:
   - `POST /api/login`
   - `GET /api/cars` (JWT protected)
3. Подключить PostgreSQL через SQLAlchemy.
4. Подготовить модели `users` и `cars`.
5. Добавить первичный сидинг администратора и тестовых автомобилей.
6. Проверить базовые сценарии: login + получение списка автомобилей.

## День 2 — Скрапер и frontend

1. Реализовать воркер для получения данных с carsensor.net.
2. Добавить retry/backoff на сетевые ошибки.
3. Сделать upsert-логику (обновление существующих, добавление новых).
4. Создать SPA:
   - `/login` (вход + сохранение JWT),
   - `/` (защищённая таблица автомобилей).
5. Подключить frontend к backend API.

## День 3 — Telegram-бот и сдача

1. Сделать Telegram-бота (aiogram).
2. Добавить обработку свободного текста через LLM Function Calling для извлечения фильтров.
3. Выполнять SQL-фильтрацию и возвращать читаемый ответ пользователю.
4. Прогнать интеграционную проверку всех сервисов через `docker compose up --build`.
5. Финализировать README и `.env.example`, подготовить репозиторий к сдаче.
